Media queries are a CSS3 module that was developed to allow content to render according to specific platforms. Media queries became standard in 2012 when browsers introduced support. They are especially popular for making adaptive layouts for mobile and web devices. We will look at making a layout that adapts to two different screen settings and mobile.

Media queries use the @media tag to separate styles within a CSS Document.

@media all and (min-width: 960px) { 
    body {
      font-size:100%}
    }
}

@media all and (max-width:780px) and (min-width:480px) {
  body {
  	font-size:80%;
  }
}

Media queries can be used to indicate the browser dimensions and resolution, device dimensions and resolution, print or web, and other attributes. It's important to also the meta tag viewport which informs the browser to look for media queries when loading content.

<meta name="viewport" content="width=device-width">

CSS3 also introduces advanced techniques for text styles, fonts, gradients, border styles and other design elements. We'll look at the CSS3 version of text-shadow and generating gradients.

CSS3 also introduces some durational effects that have previously been the domain of JavaScript and libraries like jQuery. The support for many CSS3 durational effects is not universal yet, but it will be soon. Often there are jQuery catches for browsers that don't provide full CSS3 support, but it still represents a simple and effective way to achieve simple animation and transition effects. We will look at making a short animation using the @keyframes tag, and a few different ways to use the transition property.
