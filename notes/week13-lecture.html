SVG

SVG stands for Scalable Vector Graphics. SVG uses XML markup to define graphics, which consist of shapes, text, colors and other properties. SVG can be created through code, exported from Adobe Illustrator designs or drawn in other programs. They can be very useful for the web, especially for sites that need to support various resolutions, because they render graphics using vectors, which provide resizing capability that isn't true for bitmap images like jpeg, png and gif.

SVGs can be written directly into the markup of a website.

<svg width="100%" height="200">
 <circle cx="20%" cy="100" r="60" fill="lavender" stroke="blue" />
 </svg>

SVG is also useful because it can be animated with JavaScript and be given attributes like id and class like CSS. Unlike Flash, which also support vector graphics for the web, SVG is open source and is compliant with the Document Object Model. SVG is supported by most current browsers. There are old versions of some browsers, like IE and Android, that do not support SVG.

SVG supports a number of predefined shapes and elements, including <circle>, <rect>, <ellipse>, <line>, <polyline>, <polygon>, <path>. The properties and values for SVG are similar to CSS but often use different syntax.

<svg>
  <ellipse cx="40" cy="120" rx="30" ry="40" fill="hotpink" />
  <line x1="0" y1="50" x2="400" y2="30" stroke="blue" stroke-width="4"/>
  <rect x="40" y="10" width="200" height="180" fill="lavender" fill-opacity="0.8" stroke="orange" />
  <polygon points="100,40,45,200,200,40, 180, 20"
  fill="green"/>
  <polyline points="40,40, 100, 100, 120,110, 140,150, 160, 170, 100, 80, 140, 60, 200, 200 "
  stroke="orange" stroke-width="4" fill="none"/>
  <text x="260" y="180" fill="purple">SVG is kewl!</text>
 </svg>

SVGs can also contain hyperlinks. Because SVG is written in XML, this link looks different from what we've used with HTML. We still use the anchor tag, but we need to prepend the href property with xlink:

<svg>
  <a xlink:href="svg.html" >
   <circle cx="150" cy="120" r="140" fill="lavender" stroke="lightblue" stroke-width="10" />
   <text x="130" y="100" fill="orange">Click here</text>
  </a>
 </svg>

SVG can also take styles from stylesheets, like CSS styles.

<style type="text/css">      
  .blur {
    fill:blue;
  }

  .blur:hover {
    fill:lightblue;
    cursor:pointer;
  }

  .blur:active {
    filter: url(#blur);
  }

  </style>

  <svg >
  <circle class="blur" cx="150" cy="120" r="50"  />
  <filter id="blur" >
    <feGaussianBlur stdDeviation="5" />
  </filter>
  </svg>

SVG also supports a bunch of filter effects, that look like Photoshop filters.

Another great aspect of SVG is the ability to animate different properties. This is cool to use with the shapes that we've been creating so far, but it gets really exciting when you think about the possibility of animating complex graphics drawn in Illustrator.

<svg>  
  <circle cx="100" cy="50" r="10" fill="lightblue" stroke="orange" stroke-width="20">
   <animate attributeName='r' from='2' to='80' begin='0' 
   dur='3' repeatCount='10' />
   </circle>
 </svg>

You can also use JavaScript to control animations.

<svg>
  <polyline points="0,100,10,90, 20, 130, 40, 10, 50, 13, 60, 27, 70, 164, 60, 143, 70, 29, 80, 120, 90, 100, 60, 120, 70, 140, 100, 160, 90, 180, 200, 40, 220, 200, 240, 200, 300, 40, 400, 50, 500, 80 " fill="none" stroke="lightgray" stroke-width="2"  />
  <polyline points="0,100,10,90, 20, 130, 40, 10, 50, 13, 60, 27, 70, 164, 60, 143, 70, 29, 80, 120, 90, 100, 60, 120, 70, 140, 100, 160, 90, 180, 200, 40, 220, 200, 240, 200, 300, 40, 400, 50, 500, 80 " fill="none" stroke="lightblue" stroke-width="5" stroke-dasharray="2000" stroke-dashoffset="2000">
   <animate begin="click.click" attributeName="stroke-dashoffset" from="2000" to="0" dur="10s"  />
  </polyline>
  <text x="200" y="20" id="click" fill="blue">Click</text>
</svg>

SVG files can also be embedded into HTML pages from external files. When you save an SVG from Illustrator or another program you will have more attributes that looke like this:

<?xml version="1.0" standalone="no"?>
<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN" "http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd">        
<svg width="100%" height="200" xmlns="http://www.w3.org/2000/svg" version="1.1">
<circle cx="20%" cy="100" r="40" fill="lavender" stroke="blue" />
</svg>

You can also just use the <svg> tag, which will just work most of the time.

When using an external file there are several ways to place it inside an HTML document, with various advantages and disadvantages. SVGs can be referenced with the img tag as a src attribute like a bitmap image. The disadvantage with the img tag is that it closes the source of the SVG, making it unavailable for scripting. SVG can also be embedded with the iframe tag and object tag, which both have disadvantages, so we'll focus on the embed tag, which allows scripting.

<embed src="cat.svg" type="image/svg+xml" />
