Introduction to CSS

What is CSS?

  • Cascading style sheets.
  • Used to separate design and layout descriptions from basic HTML content.
  • "Cascading" refers to the way multiple styles can be applied to one page, with different priority given to the styles.
  • Idea has existed since beginning of markup languages, "SGML" or Standard Generalized Markup Language (XML & HTML).
  • As HTML grew in popularity, web developers were frustrated at the lack of control over appearance sites had in different browsers and machines.
  • In 1994, nine different style sheet languages were proposed to the World Wide Web Consortium.
  • In 1997, CSS had its own working group.
  • CSS3 started in 1998, still used today.
  • Full compatibility with browsers around 2005.
  • CSS3 split into "modules", ie "Color".

Three ways to implement CSS

inline

  • Adding styling inside of HTML tags.
  • Least effective use of CSS.
  • Not reusable or global.
  • Okay to fix something that only happens once.
  • Many tags not supported for inline.
  • Overrides internal and external sheets.

Example:

<p style="color:red;text-decoration:line-through;background-color:white;display:inline;font-size:50px;">Hello World!</p>

Hello World!

internal style sheet

  • Written into head of HTML.
  • Not ideal, better than inline.
  • Okay for static pages.
  • Overrides external sheets.

Example:

<head> <style type="text/css"> body { margin:10%; } p { font-size:14px; } </style> </head>

external style sheet

  • The best way to implement CSS.
  • CSS rules are written into separate page, like styles.css or css.css.
  • Can be applied to multiple pages, on top of other styles.
  • Add sheet to HTML with path link in head.

Example:

<link rel="stylesheet" type="text/css" href="styles.css">

Stylesheet is a separate document that looks like this:

@charset "UTF-8";
/* CSS STYLES */

html, body {
margin:0;
padding:0;
}

....

CSS Basic Syntax

Selectors, Properties, Values

  • Selectors are names of styles.
  • Basic selectors are named after HTML tags: body, html, p, h1, etc.
  • Properties describe design/style elements: font-size, color, line-height, etc.
  • Values are numbers or descriptions of properties: 14px, red, Times New Roman, etc.
  • Format is:
    selector { property: value }
  • Very flexible for writing, will explore different ways to combine style selectors together. ie:
    h1, h2, h3 { font-family: sans-serif }

Selectors

  • Type: p, h1, body, etc.
  • Type selectors are written like this: 
    p { color: pink }
  • Class: usually describes group of divs or html tags. Written inside the html brackets: <div class="content"></div>
  • Class is written like this: 
    .content { color: red }
  • ID: usually used to describe a specific div or html element. Example: <div id="chapter1"></div>
  • ID is written like this: 
    #chapter1 { color: blue }

  • Compound selectors combine type, class and id to create styles that apply to specific elements or multiple elements. Some examples:
    • a.nav, a.content { color: blue; text-decoration: underline }
    • p#sidebar { font-size: 14px }
    • p.content #diary { font-style: italic }

Descendant

  • Describes elements that is a descendant of another element in document tree.
  • h1 strong { text-decoration:underline }
  • Used: <h1>My cat is <strong>very</strong> large.</h1>

Child

  • Difference between child and descendent is sort of confusing. Child is your direct descendent. So if there are multiple tags contained within a particular div or p, the ones effected are directly enclosed in the original div.
  • body > p { color: purple }

Sibling

  • Elements that sit next to other elements. A lot of css types have predetermined rules, like p always has a top and bottom margin. If this doesn't suit your design, you could add a sibling description to your h1 or h2 to change that.
  • h1 + p { margin-top: 0 }
  • This will only effect a p directly following an h1.

Attribute

  • Nice, easy way to add styles to whatever type you want with a specific title, not worrying about class or id.
  • [title=cats] { border: 1px solid black }
  • <p title="cats"></p>
  • Also written: 
    [title~=cats]
    or just 
    [title]
  • You don't see this method as much as others.

Universal

  • Like the name implies, these rules apply to html tags regardless of where they are nested on the page, using an asterisk.
  • *.content
  • *#user

Pseudo

  • Pseudo CSS selectors describe elements that cannot be described by the document tree, usually elements that are dynamic in some way.
  • Very useful for links, targets, etc.
  • :link
    describes links that have not been visited.
  • Other link pseudo classes include 
    :visited, :hover, & :active
  • Written like this: 
    a:hover { color: red }
