Responsive layout -- liquid & elastic

Responsive web design uses dynamic CSS styles to optimize layouts and designs for different screen sizes. It is related to but different from adaptive design, which focuses on the user experience and not the browser. Both concepts are based on the idea of progressive enhancement, which describes a process for web design that moves from simplicity to complexity, starting with basic designs and markup and leading toward complex sites. There are many concepts and methods that make up these ideas. We're going to start with liquid and elastic layouts.

Liquid and elastic layouts are based on similar principles, making content that resizes to fit the browser window that it is in. So far we have focused on making static sites that appear with the same dimensions regardless of the context a user views them in. While there are certain resolutions and dimensions that are more popular than others (check out the statistics for browser usage) these trends change over time and it is good to accommodate different users and viewing experiences.

Starting with liquid layouts, the major difference is that we use percentage values for our sizes instead of pixel numbers. Liquid layouts also use mid-width and max-width values to set the bounds of a layout.

We will make two layouts, the first using liquid and the second using elastic, with the same content.

Elastic layouts rely on em measurements to change the content size in a layout. Browsers usually default to a font size of 16px, but users can change that default value. Elastic layouts work with those values using the relative em sizing. 1em is equivalent to the inherited font size of each HTML element and can be calculated to the third decimal place. Elastic layouts require some math in order to convert designs into specific em units. For examples, with our 960px width layout, we have to divide 960 by 16 to get 60em, the width for our container. Once you get this, the rest is relatively easy.
