Web standards

Web standards are standards or web design and development that are documented by various groups, including the World Wide Web Consortium, which offer non-proprietary technical specifications for web sites. These standards are the basis for best practices, a philosophy of web design that observes the standards.

In common use, this means that a web site complies with valid HTML, CSS and JavaScript coding. It also includes semantic guidelines and accessibility standards. There are various groups dedicated to updating standards for HTML and other markup languages, CSS and stylesheets and JavaScript and other scripting languages.

In practical usage, the validator resource provided by the W3C, is basically the best way to validate code and learn best practices through experience and trial and error. Familiarizing oneself with web standards is a good way to learn to write efficient code that works.HTML Tidy is another good resource.

An example of a web standard is using the DOCTYPE declaration for the appropriate HTML standard. We have been using HTML5, which is simply <!DOCTYPE html>.

Another useful service is Link Checker. Web standards dictate that links remain valid and that broken links are removed, though this is almost never the case, and a huge fraction of legal documents, government documents and other sources online are filled with broken links.

Web standards dictate against the use of deprecated CSS tags like the FONT property or the FACE attribute. It is considered harmful.
