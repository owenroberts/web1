HTML tags have attributes, which add information for the browser, such as style, scripting and layout. An example of an attribute is the href and target in an anchor tag: <a href="google.com" target="blank" >My link</a>.

Nesting

Nesting is an important concept to understand when for when we start using the DOM. DOM manipulation requires "traversing the DOM", or moving from element to element using relationships. The html elements is always going to be the grandfather element of any element. Parent elements are the element in which an element is directly nested, like head or body in html. The elements directly inside of an element are child elements. children of those elements are the descendants of the previous element. These terms are used by JavaScript to locate specific parts of the document

CSS syntax

Basic CSS syntax looks like this: selector { property: value }. The selector refers to the HTML tag, the property describes a attribute like color or font-size, and the value sets that value. Look here to review more syntax rules.

CSS Positioning

The position property allows you to tell HTML elements where to go on a page. CSS elements default to "static" meaning they will flow with other elements in the page.

Floating

  • Pushes elements into the flow of the page, either to the right or left.
  • Best way to position elements next to each other, place images within text, and layout pages.
  • Clear: Tells the next elements not let floated elements next it.
  • Values for float and clear include: right, left, none, inherit

Fixed

  • Positions elements in browser window, regardless of other elements.
    position: fixed;
  • Can be placed with top, bottom, right, left

Relative

  • Positions relative to normal position in the flow of page.
  • Relative elements can overlap with other elements.

Absolute

  • Positions relative first parent element
  • Removed from the normal flow of the page.

Class vs. id

CSS rules are typically applied using a class selector, .myclass, or an id, #myid. Class rules are meant to be applied to many different elements on a page and can be written as compound class like this: <p class="red centered content" id="myid"> in which each word refers to a set of rules. Id can only be used once on a page and each tag can only have one id. These standards are not enforced consistently by errors, but are essential for best practices.
