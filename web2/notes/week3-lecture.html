Document Object Model

The DOM is an important API (Application Programming Interface) for JS developers, because it provides access to the structural elements of an HTML page, allowing scripting to modify the contents and design of a page. The DOM organizes parts of the page into objects, with properties and methods that can be referenced by JavaScipt (as well as other scripting languages). This allows us to create dynamic page designs, combining the HTML standard, CSS and scripting to create animation and dynamic changes to design.

Let's look at some examples:

function grafs() {
  g = document.getElementsByTagName('p');
  console.log(g);
}

Open the JavaScript console to see the results when you click the button. Then type g into the console and press Enter. g is a variable, a list of objects from the web document, in this case, paragraphs, or <p> tags. We can manipulate any HTML tag this way.

All broswers come with some implementation of the DOM that can be accessed through JavaScript or other scripting languages. The DOM can be accessed with a simple set of commands. Look at MDN's DOM reference for an easily accessible list.

Let's go over a few important DOM elements, using the console:

The window object refers to the web page itself. We can use it to determine when the page has been loaded or to make an alert.

window.alert('I am using the DOM');

document refers to our HTML page. We can use it to create and modify content.

heading = document.createElement("h1");
text = document.createTextNode("DOM Lecture!");
heading.appendChild(text);
document.body.appendChild(heading);

In this example, createElement makes a new HTML element, createTextNode makes a piece of HTML text, which is appended to the header we created with the appendChild method, and then added to the body.

The DOM uses the HTML element hierarchy to establish relationships between elements and move between them. The child element is an element directly nested into another element. descendent elements are any elements inside of the hierarchy of an element. Similarly, parent and ancestor elements are those that have elements nested inside of them. sibling elements are those that are next to one another. In the DOM, elements and attributes are referred to as nodes, hence the createTextNode. When we get DOM elements in JavaScript, like the g object with the list of paragraphs in the first example, we are using a nodeList. Let's look at the nodes in our element example.

var elem = document.getElementById('element');

We can get the children and attributes of this element.

elem.attributes;
elem.attributes.id;
elem.attributes.style;

Along with getElementById, let's look at some other important DOM methods.

var elem = document.getElementById('element');
elem.getElementsByTagName('div');
elem.innerHTML = "new text";
elem.style.color = "red";
elem.getAttribute('class');

Many other things are possible with the Document Object Model. Look at the resources to explore further.

Console errors

What happens when you enter the following line of code into the JavaScript console in Chrome?

var grafs = document.getElementByTagName('p');

By now, you have probably seen some of these errors. The console is a great resource for this kind of feedback. If you are missing a semi-colon or parenthesis, the console feedback will usually find it before you do.
