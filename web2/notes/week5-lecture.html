Arrays & Objects

Arrays

We talked about Arrays a bit last week. Today in the demo we're going to use arrays to popular the dynamic grid we're going to make with color and content. Let's go over arrays again with that in mind. Arrays in JavaScript are objects that are like lists. They are not typed, so we can put any variable, datatype, function or object into an array. The basic construction sytax for an array looks like this:

var myArray = [0, 1, 2, 3];

Arrays can also be constructed without a variable name:

new Array("zero", "one", "two");

JavaScript arrays have methods and properties that allow us to traverse and manipulate them with ease. We use index numbers and brackets to reference specific elements. The length property gives us the number of elements. Because JavaScript indexes start at 0, the last element of the array will be equivalent to length - 1.

var colorArray = ["red", "blue", "orange"];
console.log(colorArray[0]);
console.log(colorArray[1]);
console.log(colorArray[2]);

We can also add to an array using brackets with index numbers.

colorArray[3] = "hotpink";
console.log(colorArray);
colorArray.length;

We can locate elements in an array using the indexOf method:

colorArray.indexOf("orange");
colorArray.indexOf("purple");

Check out the link to MDN's Array reference to see other methods and properties of arrays.

Objects

Objects are the basic building block of JavaScript. They can be thought of like objects in the world, which have names, properties and functionalities. For instance, a train is an object. There are many different trains that have similar properties, such as color, speed and size, but different names, which can be divided into groups with common properties. We have already seen this idea in action, when we made an array and referenced the length property. This is the syntax to describe the properties of an object: myArray.length. We can create out own objects and specify what properties they have.

myObject.property

We create objects the same way we create functions or variables, and define properties by defining them.

var njTrain = new Object();
njTrain.color = "silver";
njTrain.line = "NJTransit";
njTrain.speed = "100 miles/hr";
njTrain.numPassengers = 200;

We can also set properties using bracket notation:

var metroNorthTrain = new Object();
metroNorthTrain["color"] = "blue";
metroNorthTrain["line"] = "Hudson";
metroNorthTrain["speed"] = "120 m/hr";
metroNorthTrain["numPassengers"] = 120;

Object property names can be any string, including "", or empty string. Any object property can be set or accessed with bracket or dot notation, regardless of how it was originally set.

Last week when we talked about for loops I mentioned another type of loop called for ... in which is used to iterate over the properties of an object.

for (var item in njTrain) {
  console.log(item);
}
for (var item in njTrain) {
    console.log(njTrain[item]);
}

We can create new objects with the values using this syntax:

var amtrakTrain = {
  color:"red",
  line:"East Coast",
  speed:"150 miles/hr",
  numPassengers: 300
}

Another way to make objects is with a constructor function. This is sort of like JavaScript's version of classes in other languages. Once a construcutor function is established we can make objects that will have the default properties, and also make objects that include more properties or don't have values for all of the properties. We do this by writing a function that takes the properties we want as it's arguments.

function Train(color, line, speed, numPassengers) {
  this.color = color;
  this.line = line;
  this.speed = speed;
  this.numPassengers = numPassengers;
}

Then we create objects based on this constructor:

var mtaTrain = new Train("yellow", "NQR", "50 m/hr", 100);
var pathTrain = new Train("blue", "Hoboken", "60 m/hr", 80);

The objects can be accessed and modified using dot or bracket notation. We can also add methods to objects, which are like functions, as well as some other special functions, which we'll cover in a later class.
